Standard Headers

X-DOC-SHA256: 0x... X-DOC-CID: Qm... X-DOC-SIG: ... X-DOC-REG: registry-name

Use these headers when transmitting instruments through TransitCloud.

Hash-First Workflow

1) Canonicalize JSON 2) Compute SHA-256 3) Pin artifact to IPFS → CID 4) Publish registry mapping 5) Verify independently

Public APIs

/api/registry → JSON registry mappings /api/evidence → protocol evidence + anchors

If you’re running the Next.js Developer Edition, serve these endpoints read-only for integrations.
